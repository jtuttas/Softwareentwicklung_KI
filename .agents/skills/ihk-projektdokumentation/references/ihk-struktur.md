# IHK-Dokumentationsstruktur: Formale Anforderungen und Inhalte

## Formale Vorgaben
| Kriterium | Vorgabe |
|-----------|---------|
| Gesamtumfang | mind. 15 Seiten (inkl. Deckblatt, Verzeichnisse, Anhang) |
| Hauptteil (Kap. 1–7) | 12–15 Seiten |
| Schrift | Arial 12pt |
| Zeilenabstand | 1,5-fach |
| Seitenränder | links 2,5 cm, rechts 2,0 cm, oben/unten 2,0 cm |
| Deckblatt | Pflicht – muss erste Seite sein |
| Inhaltsverzeichnis | Pflicht – direkt nach Deckblatt |
| Seitenzahlen | Pflicht (in Fußzeile) |
| Eidesstattliche Erklärung | Pflicht – auf eigener letzter Seite |
| Projektdauer | Exakt 80 Stunden |

## Pflichtkapitel und Inhalt

### 1. Projektbeschreibung (1,5–2 S.)
- Unternehmensvorstellung: Branche, Größe, Kerngeschäft
- Ausgangssituation: konkretes Problem mit Mengenangaben
- Projektziel: messbar formuliert
- Projektabgrenzung: was NICHT gemacht wird

### 2. Projektplanung (2–2,5 S.)
- Vorgehensmodell mit Begründung
- Zeitplanung: Phasentabelle Soll/Ist (Soll = genau 80h)
- Wirtschaftlichkeitsanalyse: Make-or-Buy + Kostentabelle mit €-Beträgen

### 3. Analysephase (2–2,5 S.)
- Ist-Analyse: Schwachstellen mit konkreten Zahlen (Zeit, Fehlerquote)
- Soll-Konzept: was das neue System besser macht
- Anforderungsanalyse: FA und NFA als Bullet-Listen

### 4. Entwurfsphase (2–2,5 S.)
- Systemarchitektur (Schichtenmodell, Begründung)
- Datenmodell (Entitäten, Beziehungen, Normalisierung)
- UI-Konzept (Struktur, Bedienbarkeit, Barrierefreiheit)
- **Ressourcenplanung** (Personal, Hardware, Software in Tabelle)

### 5. Implementierungsphase (3–4 S.)
- Technologieentscheidungen (mit Begründung)
- Kernkomponenten: je Unterabschnitt eine Schicht
- Herausforderungen und Lösungsansätze

### 6. Qualitätssicherung (1,5–2 S.)
- Teststrategie (3 Ebenen: Unit, Integration, System)
- Testfälle in Tabelle (ID, Beschreibung, Erwartet, Ergebnis)
- Testergebnis und Fehlerbehebung

### 7. Fazit (1–1,5 S.)
- Soll-Ist-Vergleich: Tabelle mit Abweichungsspalte
- Reflexion: Zielerreichung, Lessons Learned
- Ausblick: Folgeprojekte, Erweiterungen

## Häufige Fehler
1. Soll-Stunden ≠ 80h
2. Keine €-Beträge in der Wirtschaftlichkeit
3. Make-or-Buy fehlt oder ohne Begründung
4. Ressourcenplanung fehlt in Entwurfsphase
5. Soll-Ist-Vergleich fehlt im Fazit
6. Große Leeräume statt Fülltext
7. Eidesstattliche Erklärung fehlt
8. Glossar/Quellen ohne tatsächliche Einträge
